What are the benefits of Version Control?

Version Control Systems (VCS) are so ubiquitous that aspiring programmers may wonder why we even bother to learn all these seemingly arcane shell commands to manage a VCS like git - why can't we just save the latest and greatest build of our code? But VCS usage is common for good reason.

First and foremost, a good VCS insures that critical flaws to your code base can be rolled back to functional states. I suspect it’s all too common for developers to release code that ostensibly works, even in a test-driven environment, only to find that it introduces a whole bunch of errors (looking at you, iOS 8.0.1!) Thankfully, worst comes to worst, a faulty version can simply be rolled back to an earlier, functional version.

Additionally, a VCS allows for better workflow. Imagine you are an adept programmer working on some gigantic enterprise software solution. You have a team of 20 developers working in tandem, all across the office, on the same code base. It would be preposterous to try and edit the same documents all at once. Tossing the thumb drive around, or having your coworker delete your precious code in some cloud IDE because it was too verbose for their liking - it just couldn’t work. While you spend several hours refactoring one portion, your coworker has already created conflicting code that undermines your earlier work. It stands to reason then that every person be granted their own iteration of the software to edit simultaneously as they see fit, and then submit that work for later integration to the master branch.

How does git help you keep track of changes?

Git, a specific type of VCS, helps keep track of changes by allowing developers to comment their commits. So instead of blindly looking up past versions, developers can intelligently and systematically view the changes introduced with each commit.
