Ruby Hashes vs Javascript Objects

February 26, 2015

Ruby Hashes

A hash in Ruby is a collection of key-values pairs. Whereas a Ruby array requires an integer for indexing, a hash allows any object type. A perfect metaphor for how hashes function are to imagine a dictionary: you pass a key (e.g. the word 'Aardvark') and a value will be returned (e.g. 'a medium-sized, burrowing, nocturnal mammal native to Africa.'). Hash-like data structures are often referred to as dictionaries in other languages.

Javascript Objects

An object in Javascript (JS) is a collection of properties, or name-value pairs. If a property's value is a function, then the property is referred to as a method.

Differences?

Now that we've covered the numerous similarities between Ruby hashes and JS objects, how are they different? Perhaps the largest difference is data encapsulation of JS objects/Ruby Hashes. JS doesn't restrict access to properties, whereas Ruby has a degree of data encapsulation by not allowing simple access to internal instance variables.
