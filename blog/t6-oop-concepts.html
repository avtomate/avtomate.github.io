Object-Oriented Programming vs Functional Programming

February 19, 2015

What is Object-Oriented Programming?

Object-oriented programming (OOP) is a programming paradigm in which data structures known as objects. These objects are used to model real-world objects (such as humans, tree, and Pembroke Welsh Corgis.) Problems are broken down into smaller and smaller pieces with the intention of being simpler and readily solvable. Objects are then made to interact with one another.

Common features of OOP include classes, inheritance, encapsulation (limiting access to particular object components), and dynamic dispatch (a method invocation will be dynamically determined by the object type) and data mutability (data is readily changable.)

What is Functional Programming?

Functional programming (FP) is a programming paradigm in which functions are evaluated without changing state or manipulating data structures (in stark contrast to OOP). By minimizing state changes, program behavior is often more easier to predict and understand.

Common features of FP include higher-order functions (functions that accept other functions as their arguments), purity (side-effects are minimized, making errors less common in theory), and immutable data (data structures are copied and manipulating, but the originals are not changed.)

TL;DR Me Please

Both paradigms have different schools of thought between the relationship of data on operations on said data. For OOP, data and operations are tightly coupled, and focus is placed on these data objects interacting with one-another. For FP, data is loosely coupled to operations (functions), and the focus is on operations rather than changing the data structures. Many programming languages feature elements from several paradigms.
