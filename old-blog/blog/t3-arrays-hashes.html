Arrays and Hashes in Ruby

January 30, 2015

Data structures are an integral part of using and understanding any computer language, and Ruby is no exception. To the uninitiated, Arrays and Hashes can be rather confusing given their many similarities. But Arrays and Hashes are completely accessible once we understand their key features and what makes each unique.

Arrays

Arrays are ordered, integer-indexed collections of any object. They can contain anything from strings, to integers, to floats, to boolean values. Below is an example of an array.

our_array = ["corgis", 21, 16.34, true]

When we say arrays are ordered, that means that when and where each element is placed is significant, and will represent their index (or how we'll access them at a later time). When we say arrays are integer-indexed this means they can be accessed by their corresponding index number (e.g. the first object has index value 0). And when we say arrays are collections of any object, this means they can contain any object type, as previously noted.

Enough theory, let's see concrete examples! To create an array, we could use the literal notation of our_array = Array.new and then proceed to add elements by doing our_array[0] = "corgis" our_array[1] = 21, and so on, but that's not incredibly efficient. Instead let's instantiate (a fancy word for create) the array AND assign objects to the array in one fell swoop. This can be done by typing our_array = ["corgis", 21, 16.34, true]. Now not only has the array been created, but we've also added 4 elements to the array!

Manipulating and accessing objects is just as simple. Keeping in mind that the first element in an array is referred to as having the index 0, we can 'puts' the first object by typing puts our_array[0]. This will output a string of the first (or 0-indexed) object in our array, "corgis". How about if we want to output the fourth element? Subtract one from four, and then we'll type puts our_array[3], which will output "true".

Hashes

Hashes (often referred to as dictionaries in other languages) are collections of unique keys and their values. So whereas arrays are indexed by ascending numbers beginning at 0, hashes can be indexed by strings or symbols (more on these later). Keep in mind that all keys must be unique, whereas values can repeat. Let's now recreate our array as a hash with some more pertinent information!

First, we'll instantiate our hash by typing favorites_hash = { "dog" => "corgi", "integer" => 21, "float" => 16.34, "boolean" => true }. Now instead of accessing "corgi" by using index 0, we would use the index "dog" like so: favorites_hash["dog"]. This would then return "corgi". How fun! Note how we used the hash rocket '=>' to associate a key with a value.

Regarding when we should use an array relative to a hash, that is a value judgement. Arrays may be better utilized when order is significant, and a string as a key wouldn't be appropriate (e.g. top 5 bands ever). A hash may be better utilized when we'll want to access given values by calling a symbol or string corresponding to it (e.g. animal_names = { "cat" => "Boots", "dog" => "Rover", "rabbit" => "Bugs" }).
